## 意图确认示例

### 示例 1 — 输入："RAG和Fine-tuning怎么选"

```
问题: "你在比较 RAG 和 Fine-tuning，你的核心关注点是？"
  header: "意图确认"
  选项:
    - "我想了解各自的适用场景和边界" : 输出结构化对比，不给推荐
    - "我有具体数据和场景，需要推荐" : 输出决策建议，附带约束条件
    - "我想看一个成本/效果的量化分析" : 输出带数据支撑的分析框架
    - "其他" : 描述你的具体需求
```

### 示例 2 — 输入："帮我写个简历"

```
问题: "你说的「写简历」，核心诉求是？"
  header: "意图确认"
  选项:
    - "我需要一份可以直接用的简历模板" : 生成型，输出成品
    - "我想知道什么样的简历能拿到面试" : 理解型，输出方法论
    - "我有简历草稿，想请你帮我优化" : 分析型，诊断+改进
    - "其他" : 描述你的具体需求
```

### 示例 3 — 输入："大家都在用微服务，我们也该用吧？"

```
问题: "你提到微服务，这个问题的本质是？"
  header: "意图确认"
  选项:
    - "我想确认微服务是否适合我们团队" : 验证型，检验你已有的判断
    - "我在纠结要不要迁移到微服务" : 决策型，帮你做取舍
    - "我想全面了解微服务的利弊" : 理解型，结构化科普
    - "其他" : 描述你的具体需求
```

## 语义模式详细信号

### 意图 1: 理解
- ❶ 输入是一个名词/概念，且没附带约束条件（"什么是DDD"）
- ❷ 用"为什么"开头但不涉及具体异常（"为什么用Redis" ≠ "为什么Redis报错"）
- ❸ 期望解释而非行动

### 意图 2: 决策
- ❶ 输入包含两个或多个互斥选项
- ❷ 描述一个需要取舍的十字路口
- ❸ 没有现成方案，要做选择

### 意图 3: 生成
- ❶ 期望产出一个不存在的产物（代码/文档/方案/设计）
- ❷ 从零开始，不基于已有对象

### 意图 4: 分析
- ❶ 提供了一个已有对象（方案/数据/现状）
- ❷ 期望得到结构化判断，不是解释也不是选择

### 意图 5: 探索
- ❶ 处于开放空间，没有明确收敛方向
- ❷ 期望拓宽视野而非收窄选择

### 意图 6: 诊断/调试
- ❶ 描述一个异常行为或预期不符的结果
- ❷ 隐含"哪里出了问题"的归因需求

### 意图 7: 说服/沟通
- ❶ 目标受众不是AI而是另一个人/群体
- ❷ 核心挑战是影响他人而非获取信息

### 意图 8: 验证/确认
- ❶ 已经包含一个结论/方案/判断
- ❷ 寻求检验而非生成

### 意图 9: 规划
- ❶ 有明确起点和终点
- ❷ 期望路径设计而非方向选择

### 意图 10: 学习/习得
- ❶ 用"教"而非"告诉我"，暗示需要可练习的知识而非信息罗列
- ❷ 涉及技能习得，不只是概念理解

### 意图 11: 优化/改进
- ❶ 基于已有对象的改进，不是从零开始
- ❷ 现状"能用但不满意"，期望变得更好
- ❸ 边界规则：当用户使用"怎么优化/改进/提升" → 主意图=优化；当用户使用"为什么/怎么回事/哪里出了问题" → 主意图=诊断；当两者混合 → 优化为主，诊断为次

### 意图 12: 比较/对比
- ❶ 输入包含两个或多个可比较对象
- ❷ 未明确是"要选一个"（→决策）还是"想搞清楚"（→理解）

### 意图 13: 总结/梳理
- ❶ 输入是一堆信息/文档/内容
- ❷ 期望压缩和结构化，不是分析也不是生成

### 意图 14: 求助/救援
- ❶ 表达了困境但没有明确的下一步期望
- ❷ 真实意图可能是诊断、决策或生成——必须穿透

## 打破映射规则的详细示例

| 信号 | 动作 | 示例 |
|------|------|------|
| 输入指向"正确但没用"的答案路径 | 至少 1 个维度从批判挑战类强制选取 | "怎么提升团队效率"→默认是 system/minimalist，应加入 `adversarial` |
| 输入包含强烈但未被质疑的前提 | 必须包含 `hidden-assumption` | "AI 取代程序员"→前提是线性替代，维度必须包含 hidden-assumption |
| 输入情绪信号强烈（焦虑/愤怒/无奈） | 优先 `role-reversal` 或 `time-horizon`，不做冷分析 | "公司又要裁员了"→不用 `structure` 做冷分析 |

## 意图混淆矩阵（v2.3）

当两个意图边界模糊时，按以下规则区分：

| 易混淆对 | 区别规则 | 检测信号 |
|---------|---------|---------|
| 优化 vs 诊断 | "怎么优化/改进/提升"→优化；"为什么/怎么回事/哪里出了问题"→诊断；混合→优化主，诊断为次 | 动词方向：未来改进 vs 过去归因 |
| 对比 vs 决策 | "选哪个/值不值/该不该"→决策；"有什么区别/差在哪/异同"→对比 | 结果预期：需要选择 vs 需要理解差异 |
| 学习 vs 理解 | "教/学/入门/上手/掌握"→学习；"是什么/啥意思/怎么用"→理解 | 动作类型：技能习得 vs 概念认知 |
| 梳理 vs 分析 | "整理/总结/归纳/提炼"→梳理；"评估/怎么看/分析/方案怎么样"→分析 | 产出类型：压缩结构 vs 判断评价 |
| 求助/救援 vs 诊断 | "卡住/搞不定/救救/帮帮我"→求助； "为什么报错/排查/挂了"→诊断 | 情绪信号：困境表达 vs 归因需求 |

求助/救援穿透规则：当输入命中"卡住/搞不定/救救"，强制使用 AskUserQuestion 穿透：
```
问题: "你说的「卡住」具体是指？"
  header: "穿透确认"
  选项:
    - "方向不清，不知道选哪条路" : 决策型，输出选择框架
    - "方案不work，排查不出原因" : 诊断型，输出去障路径
    - "知道该做什么但推不动" : 执行型，输出阻力分析
    - "其他" : 描述你的具体卡点
```

## 置信度评分系统（v2.3）

### 评分公式

```
confidence_score = keyword_match * 0.3 + semantic_pattern * 0.4 + emotional_signal * 0.2 + positional_priority * 0.1
```

| 因子 | 取值 | 说明 |
|------|------|------|
| `keyword_match` | 0.0-1.0 | 命中意图关键词比例（命中n/总关键词数） |
| `semantic_pattern` | 0.0-1.0 | 语义模式匹配度（见上文"语义模式详细信号"，命中几条/总条数） |
| `emotional_signal` | 0.0-1.0 | 情绪信号强度（焦虑/愤怒/无奈=0.8，轻度困惑=0.4，无=0.1） |
| `positional_priority` | 0.0-1.0 | 用户叙事顺序（提到的第一个意图=1.0，第二个=0.7，第三个=0.4） |

### 修正因子

| 条件 | 修正 |
|------|------|
| 存在次意图 | -0.15 |
| 意图与自学习历史偏好矛盾 | -0.10 |
| 用户在上一轮同关键词模式下确认过意图 | +0.10 |
| 输入长度 < 10字 | -0.05 |

### 置信度阈值

| 级别 | 阈值 |
|------|------|
| **高 (High)** | ≥ 0.75 |
| **中 (Medium)** | 0.45 - 0.74 |
| **低 (Low)** | < 0.45 |

### 次意图权重规则

- 如果 primary_score - secondary_score ≤ 0.15，视为**共主意图**，输出时两个意图的维度混合使用
- 最大支持 2 个意图（主+次）；检测到 3 个以上时取 score 最高的 2 个
- 次意图影响：施压策略优先使用主意图的默认策略，维度提示使用两个意图的强维度混合

## 汉语语言学信号模式（v2.3）

除关键词匹配外，以下汉语语言学信号辅助意图识别：

| 模式 | 意图信号 | 权值 |
|------|---------|------|
| 吗 结尾（"好用吗/可以吗/对吗"） | 验证/确认 | +0.2 to 验证 |
| 吧 结尾（"该...吧/应该...吧"） | 验证（寻求确认） | +0.25 to 验证 |
| 呗 / 嘛 结尾（"随便呗/本来就是嘛"） | 求助/试探 | +0.15 to 求助 |
| 被 / 让 / 给（丧失控制权，"被拒"、"让老板"） | 求助/诊断 | +0.3 to 求助 |
| 看看 / 试试 / 有什么（开放探索） | 探索 | +0.2 to 探索 |
| 难道 / 岂不 / 不是吗（反问） | 说服/验证 | +0.2 to 说服 |
| 一直 / 总是 / 每次（习惯性否定） | 求助/诊断 | +0.15 to 求助 |
| 到底 / 究竟（强调追问） | 决策/验证 | +0.2 to 决策 |

用法：在计算 confidence_score 时，将命中模式的权值加到对应意图的 `keyword_match` 因子上。仅当关键词匹配得分模糊（0.3-0.7）时，语言学信号作为 tie-breaker。|
