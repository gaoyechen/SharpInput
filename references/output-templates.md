## 角色预设注入语（可复制问题的开头部分）

自动匹配逻辑见 SKILL.md「Stage 3 Role Preset」。
以下为五个角色的参数化注入语。**参数从上下文补全和自学习中提取**，如无可用参数则使用默认值。

### 参数推导规则

在生成注入语时，按以下顺序提取参数：

1. **domain**: 从用户输入的技术名词/领域术语提取（如"微服务/React/产品管理"）。如不可得，从自学习 `context_domain` 获取。仍不可得则留空。
2. **experience**: 从用户上下文推断（默认"10年以上"）。如用户明确提及年限则使用该值。
3. **skill**: 从用户意图推断（领域专家→实战/诊断类技能，攻防教练→沟通/说服类技能）。可选参数。
4. **cross_domain**: 仅跨界猎人使用。从用户输入的反方向推导（如"微服务"→跨界参考目标="单体架构/Serverless"）。

参数填写优先级：**用户输入 > 上下文补全 > 自学习记录 > 默认值**

### 参数化模板

| 角色 | 模板（`[domain]`/[experience] 为可替换参数） |
|------|----------------------------------------------|
| **领域专家** | `请以「深耕[domain]领域[experience]的资深从业者」的视角回答。不要给教科书答案，给实战结论——你见过这个领域所有的坑和反直觉的真相。` 默认：`[domain]`=该领域，`[experience]`=10年以上 |
| **魔鬼代言人** | `请以「魔鬼代言人」的视角回答——你的工作是挑战主流观点，专找共识中的漏洞和盲区。主动为反面立场辩护，找出最可能推翻结论的论据。` 如有 domain：`请以「[domain]领域的魔鬼代言人」的视角回答——...` |
| **实战操盘手** | `请以「实战操盘手」的视角回答——不问"什么是对的"，问"什么能落地"。倒推执行路径，预判真实阻力，给出本周就能动的第一个具体动作。` 如有 domain：`请以「[domain]领域的实战操盘手」的视角回答——...` |
| **攻防教练** | `请以「攻防教练」的视角回答——先站在[target]立场找出最抗拒的3个理由，再逐个设计瓦解方案。说服不是灌输，是攻防。` `[target]`=用户输入中的说服目标（如"老板/客户/团队"） |
| **跨界猎人** | `请以「跨界猎人」的视角回答——你的专长是从其他领域搬运被忽视的模式、类比和框架。至少引入一个本领域之外的参考系（如[cross_domain]领域的成熟模式）来重新审视问题。` `[cross_domain]`=从反方向推导的参照领域 |

### 完整注入示例

```
domain="微服务架构", experience="8年"
→ 请以「深耕微服务架构领域8年的资深从业者」的视角回答。不要给教科书答案，给实战结论——你见过这个领域所有的坑和反直觉的真相。

domain="React", target="老板"
→ 请以「React领域的攻防教练」的视角回答——先站在老板立场找出最抗拒的3个理由，再逐个设计瓦解方案。说服不是灌输，是攻防。

domain="后端开发", cross_domain="金融交易系统"
→ 请以「跨界猎人」的视角回答——你的专长是从其他领域搬运被忽视的模式、类比和框架。至少引入一个本领域之外的参考系（如金融交易系统的成熟模式）来重新审视问题。
```

## 通用规则

- 每次输出开头标注级别：`[Level 0]` / `[Level 1]` / `[Level 2]` / `[Level 3]`
- 结尾提示：`说「升级」可进入 Level X。`
- 语言匹配用户输入
- 用户说"stop"或"直接给问题" → 跳过剩余阶段，输出当前内容

## Level 0 — 快速施压

```
[Level 0]

> [角色注入语]
> [注入四大约束的优化问题，可直接复制]

角色预设: [领域专家/魔鬼代言人/实战操盘手/攻防教练/跨界猎人] — [1句理由]
一秒反问: [反直觉方向 — "你有没有想过 X 可能不是问题，Y 才是？"]

共识等级: [低/中/高]
说「升级」可进入 Level 1~3。
```

## Level 1 — 轻度优化

```
[Level 1]

诊断: [1-2句：核心意图 + 主要缺陷]

优化后:
> [角色注入语]
> [优化问题，可直接复制]

角色预设: [领域专家/魔鬼代言人/实战操盘手/攻防教练/跨界猎人] — [1句理由]
【演化锚】: [时间线定位]
【同类锚】: [空间参照 + 差异]

改进点: [具体改了什么]
风险提示: [来自轻量自检的脆弱假设]
说「升级」可进入 Level 2/3。
```

```

施压策略: [主策略] + [补充]

诊断: [2-3句：核心意图 + 主要缺陷 + 不优化会得到什么平庸回答]

优化后:
> [角色注入语]
> [注入四大约束 + 深度施压的优化问题，可直接复制]

角色预设: [领域专家/魔鬼代言人/实战操盘手/攻防教练/跨界猎人] — [1句理由]
【演化锚】: [时间线定位]
【同类锚】: [空间参照 + 差异]

施压详情:
- 立场约束: [强制了什么立场]
- 反共识约束: [挑战了什么主流观点]
- 取舍约束: [放弃什么换取什么]
- 可执行约束: [要求什么具体步骤]
- 深度施压: [注入了什么额外维度]

共识度: [低/中/高]

被舍弃的维度:
1. [维度]: [1句理由]
2. [维度]: [1句理由]

风险提示: [来自轻量自检的脆弱假设 + 翻转条件]
一句话警告: [不优化的最大风险]
说「升级」可进入 Level 3。
```

## Level 3 — 深度对抗

Level 3 分两阶段：**Phase 1**（分析 + 路径 + 选择）一次输出，**Phase 2**（最终输出）在用户选择后输出。

### Phase 1：分析 + 路径 + Judge 审查

**不做对比表。** 每条路径独立展示完整优化问题 + Judge 审查结果。所有审查字段来自 Judge，不得自行修改。

```
[Level 3]

诊断: [2-3句]

意图: [类型]
施压策略: [策略]

路径 A — [维度标签]:
> [角色注入语]
> [完整优化问题，可直接复制]
【演化锚】: [时间线定位]
【同类锚】: [空间参照 + 差异]
反方最强攻击: [Judge 报告摘要，1句话]
真实反例: [案例] 或 未验证
翻转条件: [条件] 或 边界模糊
风险判定: [可靠 / 有条件 / 高风险]

路径 B — [维度标签]:
> [角色注入语]
> [完整优化问题]
【演化锚】: [时间线定位]
【同类锚】: [空间参照 + 差异]
反方最强攻击: [摘要]
真实反例: [案例] 或 未验证
翻转条件: [条件] 或 边界模糊
风险判定: [可靠 / 有条件 / 高风险]

路径 C — [维度标签]:
> [角色注入语]
> [完整优化问题]
【演化锚】: [时间线定位]
【同类锚】: [空间参照 + 差异]
反方最强攻击: [摘要]
真实反例: [案例] 或 未验证
翻转条件: [条件] 或 边界模糊
风险判定: [可靠 / 有条件 / 高风险]

被舍弃的维度:
1. [维度]: [1句理由]
2. [维度]: [1句理由]
```

文本输出完成后，用 AskUserQuestion 让用户选择：
```
问题: "选择路径（可多选），我会输出最终打磨好的问题"
  header: "选择路径"
  multiSelect: true
  选项:
    - "A — [维度标签]" : 风险判定: [可靠/有条件/高风险] | 反例: [摘要]
    - "B — [维度标签]" : 风险判定: [可靠/有条件/高风险] | 反例: [摘要]
    - "C — [维度标签]" : 风险判定: [可靠/有条件/高风险] | 反例: [摘要]
```
- 最低风险路径排第一（推荐提示）
- 不需要"组合"选项——多选本身支持组合

### Phase 2：最终输出

**单选：**
```
已选择: [路径 X — 维度标签]

优化后的问题:
> [角色注入语]
> [最终打磨问题，可直接复制]

角色预设: [领域专家/魔鬼代言人/实战操盘手/攻防教练/跨界猎人] — [1句理由]
施压约束:
- 立场 / 反共识 / 取舍 / 可执行

适用边界: [最佳使用条件]
风险提示: [来自 Judge 的反例或翻转条件]
如果方向不对，追问: "[追问问题]"
```

**多选组合：**
```
已组合: [路径 X + Y]

融合策略: 以 [X] 为基础，注入 [Y] 的 [要素]

优化后的问题:
> [角色注入语]
> [融合后的优化问题]

角色预设: [领域专家/魔鬼代言人/实战操盘手/攻防教练/跨界猎人] — [1句理由]
施压约束:
- 立场 / 反共识 / 取舍 / 可执行

适用边界: [条件]
风险提示: [反例或翻转条件]
如果方向不对，追问: "[追问问题]"
```

组合规则：
- 2 条路径：低风险路径为基础，注入另一条的要素
- 3 条全选：最低风险为基础，提取其他两条核心要素（标注「三条全组合可能导致约束过多，已提取核心要素」）
- 要素冲突时用冲突决策优先级（安全 > 信息量 > 用户偏好），标注冲突并说明优先理由

## 常用对话模板

#### 预算确认
```
问题: "你的预算范围是？"
  header: "预算"
  选项:
    - "1000-2000 元" : 入门级选择
    - "2000-5000 元" : 中端选择
    - "5000-9000 元" : 高端选择
    - "其他" : 自定义预算
```

#### 时间确认
```
问题: "你期望的时间范围是？"
  header: "时间"
  选项:
    - "1周内" : 紧急
    - "1个月内" : 中等节奏
    - "3个月+" : 不急
    - "其他" : 自定义时间
```

#### 规模确认
```
问题: "你的问题规模是？"
  header: "规模"
  选项:
    - "具体问题" : 单一明确
    - "系统性问题" : 多关联因素
    - "战略性问题" : 长期大方向
    - "其他" : 自定义
```
