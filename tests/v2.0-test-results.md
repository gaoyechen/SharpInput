## 核心问题 & 修复建议

### P0: 优先级矩阵硬规则与语义冲突

**问题**: TC-09 和 TC-10 暴露了同一个根本问题——优先级矩阵的硬性排序会翻转用户的真正意图。

- TC-09: 用户说"卡住了"（求助），矩阵强制诊断为主
- TC-10: 用户说"该用吧+帮我写方案"（验证→生成），矩阵强制生成为主

**修复方案**:
1. 矩阵降级为**参考权重**而非硬规则
2. 增加语义信号和情绪信号的修正因子
3. 当语义分析置信度 > 矩阵推断时，语义优先
4. 对于"验证+生成"组合，允许保序（按用户叙事顺序）

**修改位置**: SKILL.md 复合意图处理部分

### P1: 优化 vs 诊断边界规则化

**问题**: "API响应时间从200ms涨到了2s，怎么优化"——表面是优化，但隐含需要诊断。

**修复方案**:
增加明确的边界规则:
```
当用户使用"怎么优化/改进/提升" → 主意图=优化
当用户使用"为什么/怎么回事/哪里出了问题" → 主意图=诊断
当两者混合（退化事件+改进请求）→ 优化为主，诊断为次
```

**修改位置**: SKILL.md 意图分类表第 11 行（优化/改进）增加边界说明

### P2: Level 0 与意图上下文的处理优先级

**问题**: Level 0 应跳过上下文问题直接输出，但某些意图（如验证）要求问上下文。

**修复方案**:
在上下文补全部分增加明确规则:
```
Level 0 跳过上下文补全，直接输出四大约束+反问。意图专属上下文仅在 Level ≥ 1 时生效。
```

**修改位置**: SKILL.md 上下文补全部分

### P2: 学习意图的 Gate 边界

**问题**: "我想学React，零基础，怎么入门"被判定为 L1，但暗示需要系统规划，可能更接近 L2。

**修复方案**:
在 Gate 决策树中增加学习意图的特殊处理:
```
如果意图为「学习/习得」且用户描述了基础水平（如"零基础"）或明确要求系统规划 → Level 2
```

**修改位置**: SKILL.md Gate 分级部分

## 结论

v2.0 的核心改动（14 类意图 + 口语关键词 + 语义模式 + 意图驱动上下文补全）**全部生效**，整体表现达到 A 级。

主要改进方向:
1. **优先级矩阵需要软化** — 从硬规则降级为参考权重，增加语义修正
2. **优化/诊断边界需要规则化** — 增加明确的判定规则
3. **Level 0 的上下文处理需要明确** — 跳过补全直接输出
4. **Context 选项精准度需要提升** — 增加输入内容感知
